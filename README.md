## Overview
This repository hosts a collection of low-level engineering projects and benchmarks designed to explore the physical reality of software execution. The goal of this portfolio is to demonstrate "Mechanical Sympathy"â€”the ability to write higher-level software that respects the underlying constraints of the hardware (CPU Caches, Memory Hierarchy, and Data Representation).

Unlike typical application-layer portfolios, this stack focuses on:
* **Zero-Dependency Implementations:** Building core utilities from scratch in C to understand the "Physics of Software."
* **Performance Benchmarking:** Empirical verification of theoretical concepts (e.g., Cache Locality).
* **Robust Systems Programming:** Writing safe, validated, and efficient code that interacts directly with memory.
